================================================================================
PARTNER WORKFLOW (after git clone):
================================================================================

1. Clone repo on Raspberry Pi:
   git clone https://github.com/smit4351/ece595_testing.git
   cd ece595_testing

2. Build kernel modules:
   bash BUILD_ON_PI.sh

3. Setup automation:
   cd pi_attack_runner
   sudo bash partner_setup.sh

4. Run attacks:
   sudo bash run_attacks.sh --local ~/ece595_testing/kernel_modules

5. Collect results:
   sudo bash collect_results.sh --output ~/attack_results

================================================================================
WHAT HAPPENS ON PI:
================================================================================

BUILD_ON_PI.sh will:
  1. Install build-essential, linux-headers
  2. Compile all 4 .ko modules natively
  3. Create: dma_attack.ko, smc_fuzzer.ko, cache_timing_attack.ko, 
            peripheral_isolation_test.ko

run_attacks.sh will:
  1. Load each .ko module
  2. Execute attack via /proc interface
  3. Monitor for crashes
  4. Collect dmesg logs
  5. Generate summary report
